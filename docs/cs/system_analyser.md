#### 系统设计（面向对象设计，设计类分类，活动图/流程图区别，状态图填充）

设计类的类型、功能：

- 实体类：映射需求中的每个实体，需要保存在永久存储中的信息；
- 控制类：控制用例工作的类，用于对用例所特有的行为进行建模；
- 边界类：用于封装在用例内、外流动的信息或数据流； 

活动图和流程图的主要区别：

- 活动图是面向对象，流程图是面向结过程；
- 活动图着重表现系统的行为，流程图着重表现系统的处理过程；
- 活动图能够表示并发活动的情形，流程图主要结构是顺序、循环、分支，严格的顺序和时间关系，不能表示并发。
